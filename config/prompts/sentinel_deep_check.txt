## Instructions

Review the diff and answer these questions:

1. **Completeness**: Does the diff fully solve the task? List any missing requirements.
2. **Correctness**: Are there logic errors, off-by-one errors, or incorrect assumptions?
3. **Edge Cases**: What edge cases are NOT handled that should be?
4. **Security**: Any injection, traversal, or credential exposure risks?

## Required Output Format (JSON)

```json
{
  "verdict": "PASS" | "FAIL",
  "gaps": ["list of missing requirements or issues"],
  "severity": "none" | "low" | "medium" | "high" | "critical"
}
```

If the code correctly and completely solves the task with no significant issues,
return verdict "PASS" with an empty gaps list. Otherwise return "FAIL" with
specific, actionable gap descriptions.
